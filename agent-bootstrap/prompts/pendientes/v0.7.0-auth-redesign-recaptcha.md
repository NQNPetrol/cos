## Descripción

Las cuatro vistas de autenticación (login, register, forgot-password, reset-password) tienen diseños
inconsistentes entre sí y con el resto de la app: login tiene un rediseño dark parcial con el logo
roto; las otras tres siguen con el diseño Jetstream blanco por defecto.

Esta versión unifica y moderniza todas las vistas auth con el sistema dark del proyecto, usa el logo
correcto de CyH Sur SA, agrega reCAPTCHA v2 al login y garantiza responsive en todas las vistas.

**Problema concreto:** `logo-cos.svg` referenciado en login.blade.php no existe → imagen rota.


## PASO 4 — Validar reCAPTCHA en el backend

El controlador de login de Jetstream pasa por `LoginResponse`. La validación del reCAPTCHA se agrega en un `FormRequest` o mediante middleware.

La forma más limpia con este paquete: agregar la validación en `App\Http\Requests\Auth\LoginRequest.php` si existe, o interceptar en un middleware.

Verificar si existe `app/Http/Requests/Auth/LoginRequest.php`:
- **Si existe:** agregar `'g-recaptcha-response' => ['required', new ReCaptcha]` en las reglas
- **Si no existe:** crear un middleware `ValidateRecaptcha` que valide el campo en cada request POST a `/login`

Método con middleware (más seguro con Jetstream):
```php
// app/Http/Middleware/ValidateRecaptcha.php
public function handle(Request $request, Closure $next): Response
{
    if ($request->isMethod('post') && $request->routeIs('login')) {
        $validator = validator($request->all(), [
            'g-recaptcha-response' => ['required', new \Biscolab\ReCaptcha\Rules\ReCaptcha],
        ]);
        if ($validator->fails()) {
            return back()->withErrors(['g-recaptcha-response' => 'Por favor completá el reCAPTCHA.'])->withInput();
        }
    }
    return $next($request);
}
```

Registrar el middleware en `bootstrap/app.php` o `app/Http/Kernel.php` según la versión de Laravel 12.

**Verificación:** Al intentar hacer login sin completar el reCAPTCHA → error "Por favor completá el reCAPTCHA." bajo el widget.
