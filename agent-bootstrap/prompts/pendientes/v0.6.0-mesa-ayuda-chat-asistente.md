**VERSIÓN:** v0.6.0  
**SLUG:** mesa-ayuda-chat-asistente  
**DEPENDENCIAS:** ninguna (recomendable tener v0.4.0 y v0.5.0 para que las guías reflejen el flujo actual)  
**ESTIMACIÓN:** 1–2 días  
**PRIORIDAD:** media-alta

---

## Descripción

El cliente pidió una **mesa de ayuda** con un **chat tipo asistente/bot** que guíe a los usuarios en el uso de la plataforma: cómo crear un ticket nuevo, cómo crear un evento, cómo registrar documentación de las patrullas, etc.

Objetivo: ofrecer una pantalla o panel "Mesa de ayuda" / "Ayuda" (accesible desde el layout cliente y opcionalmente admin) con una interfaz de chat donde el usuario escribe o elige temas y el asistente responde con pasos concretos y, cuando aplique, enlaces a la pantalla correcta (crear ticket, crear evento, documentación patrullas). Las respuestas pueden ser estáticas (FAQ / árbol de temas) o con detección simple por palabras clave; no es obligatorio integrar IA/LLM externos.

## PASO 1 — Ruta y entrada a la mesa de ayuda

- Crear una ruta (ej. `client/ayuda` o `client/mesa-ayuda`) que muestre la vista del chat/asistente, protegida por auth y permiso de acceso al área cliente (o mismo middleware que el resto del layout cliente).
- Agregar en el menú o layout cliente un enlace visible "Ayuda" o "Mesa de ayuda" que apunte a esa ruta.
- Crear la vista base: título "Mesa de ayuda" o "¿En qué podemos ayudarte?" y contenedor para el chat (área de mensajes + input o botones de temas).

**Verificación:** Usuario cliente puede abrir la mesa de ayuda desde el menú y ve la pantalla del chat.

---

## PASO 2 — Interfaz de chat y respuestas del asistente

- Implementar la UI de chat: mensajes del usuario (texto o selección de tema) y respuestas del asistente en burbujas o bloques diferenciados.
- Definir un conjunto inicial de temas/respuestas:
  - **Crear un ticket nuevo:** pasos breves (ir a Tickets → Nuevo ticket, completar título, categoría, descripción, enviar) y enlace a la ruta de creación de tickets.
  - **Crear un evento:** pasos breves y enlace a la ruta de creación de eventos (cliente).
  - **Registrar documentación de patrullas:** pasos breves y enlace a la sección correspondiente (patrullas / documentación).
- Las respuestas pueden ser estáticas: por ejemplo, un select o botones "Crear ticket", "Crear evento", "Documentación patrullas", "Otro" y al elegir mostrar el texto + enlace. Opcional: detectar palabras clave en un input de texto libre y mostrar la respuesta correspondiente.
- Incluir al menos un fallback: "No encontré eso. Podés elegir: Crear ticket, Crear evento, Documentación patrullas."

**Verificación:** Usuario puede elegir "Crear ticket" (o escribir algo similar) y recibe los pasos y el enlace; igual para evento y documentación patrullas.
