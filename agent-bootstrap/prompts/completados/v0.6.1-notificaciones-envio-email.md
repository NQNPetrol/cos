## Descripción

Agregar la posibilidad de enviar por correo electrónico las notificaciones creadas desde la administración. Dos puntos de entrada:

1. **Al crear:** checkbox "Enviar también por email" en el formulario de creación; si está tildado, tras guardar la notificación se dispara el envío.
2. **Desde el listado:** botón "Enviar email" en la columna de acciones de cada notificación, para (re)enviar cuando se quiera.

Destinatarios según el tipo de notificación:
- `global` → todos los usuarios activos del sistema
- `user` → el usuario seleccionado (`user_id`)
- `client` → todos los usuarios vinculados a ese cliente (tabla `user_cliente`)

El correo es simple: asunto = título de la notificación, cuerpo = mensaje de la notificación. Sin plantillas elaboradas.

## PASO 2 — Método de envío en el controlador

En `NotificationController.php`, crear un método privado `sendNotificationEmail(Notification $notification)`:

1. Determinar destinatarios según `$notification->type`:
   - `global`: `User::all()` (o filtrar por activos si hay campo `is_active` en users).
   - `user`: `User::find($notification->user_id)` — un solo destinatario.
   - `client`: usuarios vinculados al cliente → `UserCliente::where('cliente_id', $notification->client_id)->pluck('user_id')` → `User::whereIn('id', $userIds)->get()`.
2. Enviar con `Mail::to($user)->queue(new NotificationEmail($notification))` para cada destinatario (usar queue para no bloquear). Si el proyecto no tiene queue configurado, usar `Mail::to(...)->send(...)`.
3. Retornar la cantidad de emails enviados para el mensaje flash.

**Verificación:** El método existe y se puede invocar desde `store()` y desde una nueva ruta.

## Notas

- No se agrega campo `email_sent` ni tracking de envío al modelo. Si en el futuro se quiere saber si ya se envió, se puede agregar como versión separada.
- Si el proyecto no tiene queue configurado (`QUEUE_CONNECTION=sync`), los emails se envían sincrónicamente. Con muchos usuarios globales puede ser lento — se recomienda configurar queue en el futuro.
- Referencia de Mailables existentes: `TicketCreatedNotification`, `AlertaNotificationMail`, `CoberturaRechazadaMail`.
