## Descripción

El administrador paga varios servicios de un mismo proveedor (varias facturas diferentes) en una sola transferencia. Hoy tiene que subir el comprobante de pago uno por uno a cada registro. Se necesita una funcionalidad de carga masiva: seleccionar múltiples pagos pendientes, subir un único comprobante de pago, y que el backend lo asigne a todos los registros seleccionados marcándolos como "pagado".

El archivo se guarda **una sola vez en disco** y se referencia desde múltiples registros (mismo `comprobante_pago_path` para todos).

## PASO 1 — Checkboxes de selección en tabla de pendientes

En `resources/views/rodados/pagos.blade.php`, en la tab "Pagos por Realizar":

1. Agregar una columna de checkbox al inicio de la tabla (thead y tbody):

   - **thead:** checkbox "Seleccionar todos" que selecciona/deselecciona todos los registros **visibles** (respetando el filtro de búsqueda activo).
   - **tbody:** checkbox por fila con `value="{{ $pago->id }}"` y una clase CSS común (ej. `pago-checkbox`).

2. El checkbox "Seleccionar todos" solo marca los rows que no estén ocultos por el filtro de búsqueda (la función `buscarPago()` oculta rows). Así el usuario puede: filtrar → seleccionar todos los filtrados → adjuntar.

3. Agregar JavaScript para:
   - Toggle de "seleccionar todos" (solo visibles).
   - Conteo de seleccionados.
   - Cuando hay ≥1 seleccionado, mostrar un **botón flotante o barra de acción** con: "Adjuntar comprobante en batch (N seleccionados)".

**Verificación:** Los checkboxes aparecen. "Seleccionar todos" selecciona solo los visibles. El botón de batch aparece/desaparece según la selección. El conteo es correcto.

## PASO 3 — Ruta y método backend para batch

En `routes/web.php`, agregar la ruta dentro del grupo de rodados:

```php
Route::post('/pagos/adjuntar-comprobante-batch', [PagoServiciosRodadoController::class, 'adjuntarComprobanteBatch'])
    ->name('pagos.adjuntar-comprobante-batch');
```

En `PagoServiciosRodadoController.php`, nuevo método `adjuntarComprobanteBatch(Request $request)`:

1. **Validación:**
   ```php
   $request->validate([
       'comprobante_pago' => 'required|file|mimes:pdf,jpg,jpeg,png|max:10240',
       'pago_ids' => 'required|array|min:1',
       'pago_ids.*' => 'exists:pago_servicios_rodados,id',
   ]);
   ```

2. **Guardar el archivo una sola vez:**
   - Store en `rodados/comprobantes-batch/` (o una carpeta compartida) con nombre único.
   - Obtener el `$comprobantePath`.

3. **Actualizar todos los registros seleccionados:**
   ```php
   PagoServiciosRodado::whereIn('id', $request->pago_ids)
       ->update([
           'comprobante_pago_path' => $comprobantePath,
           'estado' => PagoServiciosRodado::ESTADO_PAGADO,
           'fecha_pago' => now()->toDateString(),
       ]);
   ```

4. **Si algún pago tiene `turno_rodado_id`**, actualizar también el `comprobante_pago_path` del turno asociado (misma lógica que el upload individual).

5. Redirect con mensaje flash: "Comprobante adjuntado a N pagos. Marcados como pagados."

**Verificación:** Seleccionar 3 pagos, subir comprobante → los 3 pasan a estado "pagado", los 3 apuntan al mismo archivo en disco, aparecen en tab "Pagos realizados".

## Notas

- No se crea un modelo nuevo para comprobantes compartidos; se reutiliza `comprobante_pago_path` existente. Si en el futuro se necesita trackear qué pagos comparten comprobante, se puede agregar un `comprobante_batch_id`.
- El archivo se guarda en `rodados/comprobantes-batch/` para diferenciarlo de los individuales que van en `rodados/{rodado_id}/comprobantes/`.
- Si el proyecto tiene configurado el fix de `client_max_body_size` (v0.6.2), los uploads grandes funcionarán. Si no, aplica el mismo 413.
